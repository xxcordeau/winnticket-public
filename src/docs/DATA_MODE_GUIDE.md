# 데이터 모드 가이드

## 📌 개요

더미 데이터와 API 호출이 섞여서 혼란스러운 문제를 해결하기 위해 **데이터 모드 설정**을 도입했습니다.

## 🎯 데이터 모드 종류

### 1. API_ONLY (기본값, 추천)
- **오직 실제 API만 사용**
- API 성공 시: API 데이터 사용
- API 실패 시: 빈 배열 반환
- **운영 환경에 적합**

### 2. API_WITH_FALLBACK
- **API 우선, 실패 시 더미 데이터**
- API 성공 시: API 데이터 사용
- API 실패 시: 더미 데이터 사용
- **개발 환경에 적합**

### 3. DUMMY_ONLY
- **오직 더미 데이터만 사용**
- API를 호출하지 않음
- **테스트 환경에 적합**

## ⚙️ 설정 방법

`/lib/data-mode.ts` 파일에서 `CURRENT_DATA_MODE`를 변경하세요:

```typescript
// 운영 환경 (기본값)
export const CURRENT_DATA_MODE: DataMode = 'API_ONLY';

// 개발 환경 (API 실패 시 더미 데이터)
export const CURRENT_DATA_MODE: DataMode = 'API_WITH_FALLBACK';

// 테스트 환경 (더미 데이터만)
export const CURRENT_DATA_MODE: DataMode = 'DUMMY_ONLY';
```

## ⚠️ 주의사항

### API_ONLY 모드 (현재 기본값)
- API가 실패하면 **빈 화면**이 표시됩니다
- 상품이 하나도 표시되지 않으면 **API 서버 상태를 확인**하세요
- 콘솔에서 `❌ [상품관리] API 응답 없음` 메시지를 확인하세요

### 더미 데이터가 필요한 경우
1. `/lib/data-mode.ts`에서 모드를 `API_WITH_FALLBACK`으로 변경
2. 브라우저를 새로고침
3. API 실패 시 자동으로 더미 데이터가 표시됩니다

## 🔧 문제 해결

### 상품이 하나도 표시되지 않을 때

1. **콘솔 로그 확인**
   ```
   📦 [상품관리] API 응답: 실패
   ⚠️ [상품관리] API 응답 없음 - 빈 배열 설정
   ```

2. **API 서버 상태 확인**
   - API 서버가 실행 중인지 확인
   - 네트워크 탭에서 API 요청 상태 코드 확인

3. **임시 조치: Fallback 활성화**
   ```typescript
   // /lib/data-mode.ts
   export const CURRENT_DATA_MODE: DataMode = 'API_WITH_FALLBACK';
   ```

### 더미 데이터와 API 데이터가 섞여서 표시될 때

- `API_ONLY` 모드를 사용하면 **절대 섞이지 않습니다**
- API가 성공하면 API 데이터만 사용
- API가 실패하면 빈 배열만 반환

## 📝 변경 사항 요약

### 제거된 코드
- ❌ 상품 관리 페이지에서 더미 데이터 직접 import
- ❌ API와 더미 데이터를 병합하는 복잡한 로직
- ❌ 불필요한 콘솔 로그 (HTML 응답 에러 등)

### 추가된 기능
- ✅ 명확한 데이터 모드 설정 (`/lib/data-mode.ts`)
- ✅ 깔끔한 콘솔 로깅 (성공/실패 명확하게 표시)
- ✅ 이미지 URL 자동 변환 (`getImageUrl` 함수 사용)
- ✅ API 우선, 명확한 fallback 로직

## 🎉 결과

- **명확한 데이터 흐름**: API 또는 더미 중 하나만 사용
- **깔끔한 콘솔**: 필요한 정보만 표시
- **쉬운 디버깅**: 어떤 데이터를 사용하는지 한눈에 확인
- **운영 환경 대비**: API만 사용하는 깔끔한 구조
